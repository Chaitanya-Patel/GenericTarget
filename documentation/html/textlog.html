Protocol File

If the target application is started, the console output is redirected to a text file. All protocol files are stored in the protocol/ directory in the software directory of the target application. A protocol file contains information about the target software such as warnings and error messages. All messages are time stamped. If the application is started with [...] --console, the output is not redirected to the text file.

Example Protocol File

Using the MATLAB class GT.GenericTarget the current protocol file is displayed: 
target.ShowLatestProtocol();    % target is a GT.GenericTarget object
The content of a protocol file can look like this:



Section 1

This area displays application information. It shows information about the generic target version, the name of the deployed simulink model, when it was compiled, when it was started, and so on. Some detailed information about the target model, such as priorities, sampletimes, etc., are also shown.

Section 2

The type of message is displayed at the beginning of a line. No character here means that it is information. An 'E' means error, a 'W' means warning. In this example, there're no errors or warnings.

Section 3

Specifies the timestamp. This time refers to the start time of the application.

Section 4

Messages during the initialization phase.

Section 5

The application has received a termination message. In this case the application was stopped by calling the Stop() member function of the GT.GenericTarget MATLAB class.

Section 6

The termination phase of the application.
