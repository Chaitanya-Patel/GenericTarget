Thread Overview

There are two threads that are always present. These are the main thread and the BaseRateScheduler thread.



The BaseRateScheduler thread has the highest priority (99). This thread waits for a timer event, which is set periodically with the base sample rate. Then this thread distributes the event to all threads of the model.

The main thread executes the main entry function of the application. The initialization and termination phases of the generic target application run in this thread. In the actual main loop this thread receives messages via the application socket and waits for a termination message (further information about the main thread).

The number of additional threads depends on the Simulink model. For each sample rate, one additional thread is added. The maximum priority is set with the MATLAB class GT.GenericTarget using the parameter upperThreadPriority. One additional thread is added for each Log To File block. The priority is specified with the parameter priorityLog via the GT.GenericTarget MATLAB class. For each UDP port (UDP Receive, UDP Send, Multicast UDP Receive or Multicast UDP Send block) an additional thread is created. The priority is set via the block mask of the corresponding block. Only one receive block but several send blocks can be used for one UDP port. If different priorities are specified for a UDP port, the highest value is used.
