Text Log

While the application is running, specific information is printed to the console. This information includes warnings and error messages. All messages are time stamped. If the application is started with [...] &> log.txt, the output is redirected to the text file log.txt.

Example Log File

Using the MATLAB class GT.GenericTarget the current log file can be displayed: 
gt.ShowLog();    % gt is a GT.GenericTarget object
The log file must be in the bin/ directory and named log.txt. A log file can look like this:



In this example, the generic target application was run on a PC with a Windows operating system without realtime.

Section 1

This area displays application information. The application is called GenericTarget and the version number is 20200329. The generic target framework was compiled on the target at 11:27:01 local time on March 30, 2020 and the model was compiled on the target at 11:29:12 on March 30, 2020. The start time of the application is March 30, 2020, 14:53:18 UTC. No further arguments were passed to the application.

Section 2

The type of message is displayed at the beginning of a line. No character here means that it is information. An 'E' means error, a 'W' means warning. It can be seen that the thread priorities cannot be set. In this case it is because the application was running on a Windows PC without realtime. On a Linux-OS with PREEMPT_RT patch and appropriate authorization (sudo) this error does not occur.

Section 3

Specifies the timestamp. As long as the model is not running, this time (in seconds) refers to the start time of the application. Only after initialization this time refers to the start time of the model.

Section 4

Messages during the initialization phase.

Section 5

The application has received a termination message. In this case the application was terminated in the console with Ctrl + C ("Caught signal 2!").

Section 6

The termination phase of the application.
